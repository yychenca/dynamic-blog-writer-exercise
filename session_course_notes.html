An AI Agent

The Brain (LLM): This is the Large Language Model (like Gemini). Its job is to reason, understand language, and create plans. It's the strategist.

The Tools (Actions): These are the agent's "hands." They are functions or APIs that allow the agent to interact with the outside world, like searching the web, using a calculator, or reading a file. They are the executors.

The ReAct Framework: How an Agent "Thinks"

An agent doesn't solve a problem all at once. It uses a step-by-step reasoning process called **ReAct**, which stands for **Reason** and **Act**. This is a continuous loop that guides the agent to a solution.

Thought: The agent first reasons about the problem and its goal. It thinks, "What is the next logical step I need to take?"

Action: Based on its thought, the agent decides to act by choosing one of its available tools and providing an input for it.

Observation: The agent receives the result from the tool. This new information is its observation.

The agent then takes this observation and starts the loop again with a new **Thought**. This continues until it has enough information to give a final answer.

From Agents to Workflows

While a single agent is powerful, the true potential is unlocked when you combine multiple agents into a **workflow**. An agent is a single worker; a workflow is a team of specialists. Here are some common patterns.

Prompt Chaining (Assembly Line)

The output of one agent becomes the input for the next. This is for tasks that must be done in a specific sequence.

Routing (Post Office)

A "dispatcher" agent analyzes a task and sends it to the most appropriate specialized agent (e.g., a Math Agent or a History Agent).

Parallelization (Kitchen Crew)

A main task is split into sub-tasks that multiple agents can work on at the same time. Their results are combined at the end.

Evaluator-Optimizer (Writer & Editor)

One agent creates a draft, and another agent critiques it, providing feedback. This creates a loop of continuous improvement. **This is the pattern for your capstone project.**

The Toolkit for Our Lab

For our hands-on lab, we will give our agent one powerful tool: the ability to search the web. We will use the **DuckDuckGo Search** tool because it's effective and doesn't require a separate API key, making it perfect for getting started quickly.

Hands-On Lab: Build a Research Agent

In the lab, you will apply these concepts to build a real agent in Google Colab. You will:

  • Initialize the Gemini 2.5 Flash model as the "Brain."
  • Define the DuckDuckGo Search tool as the "Tool."
  • Combine them using LangChain to create an agent.
  • Run the agent and watch its ReAct thought process live as it answers a question.
